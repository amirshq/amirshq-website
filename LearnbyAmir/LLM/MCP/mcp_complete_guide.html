What is MCP?

Model Context Protocol (MCP) is a universal connector that lets AI assistants talk to tools, databases, and services.

Simple Analogy

Think of MCP as a USB-C port for AI.

Without USB-C: Every device needs a different cable. Connect a phone, then a laptop, then a camera—different plugs for each.

With USB-C: One cable works for everything. Plug it in, and it automatically works.

MCP is the same: Claude doesn't need custom code for Qdrant, PostgreSQL, APIs, etc. One MCP system connects to anything.

Real-World Examples

✅ Database Access

MCP → PostgreSQL → Get customer data

🔍 Vector Search

MCP → Qdrant → Semantic search documents

🖼️ Image Analysis

MCP → Computer Vision → Analyze images

🌐 API Calls

MCP → REST API → Fetch data

What You Get with MCP

  • Universal interface - Same format for all tools
  • No vendor lock-in - Works with Claude, ChatGPT, Gemini
  • Framework agnostic - LangGraph, AutoGen, CrewAI all supported
  • Easy to maintain - One MCP = multiple applications
  • Scalable - Add tools without rewriting code
Key Insight: MCP is about standardization. Instead of writing custom code for each tool, you write one MCP that works everywhere.

The Problem MCP Solves

Before MCP: The Mess

Imagine you're building an AI app that needs to talk to 5 different systems:

Without MCP
Your App
    ├─ Custom code for PostgreSQL (50 lines)
    ├─ Custom code for Qdrant (50 lines)
    ├─ Custom code for Pinecone (50 lines)
    ├─ Custom code for REST API (50 lines)
    └─ Custom code for Redis (50 lines)

Total: 250 lines of custom integration code

Change tools? Rewrite everything.
Use with LangGraph? Add 50 more lines.
Use with AutoGen? Add 50 more lines.
Use with CrewAI? Add 50 more lines.

Result: 350+ lines of custom code just to connect tools.

After MCP: The Solution

With MCP
Your App
    ├─ PostgreSQL MCP (100 lines, write once)
    ├─ Qdrant MCP (100 lines, write once)
    ├─ Pinecone MCP (100 lines, write once)
    ├─ REST API MCP (100 lines, write once)

Add a 6th tool? Write 100 lines for new MCP.
Change tools? Just swap MCPs, no app code changes.
Use with LangGraph? Works automatically.
Use with AutoGen? Works automatically.
Use with CrewAI? Works automatically.

Result: 100 lines per tool + MCP, then it works everywhere.

The Key Difference

The Problem in One Sentence

Without MCP: You write custom code for every tool × every framework = exponential complexity.

With MCP: You write one MCP per tool, and it works with every framework.

MCP Architecture

How MCP Works (3 Parts)

1
MCP Client

The part that asks for help. Usually your Claude Desktop, Claude API, or AI agent (LangGraph, AutoGen).

2
MCP Server

The part that does the work. You build this. It connects to your database, API, or service.

3
Connection

The protocol that connects client to server. Usually JSON-RPC over stdio or HTTP.

The Flow

MCP Request/Response
User: "Find customers named John"
    ↓
Claude (Client): "I need to call search_database tool"
    ↓
MCP Client → MCP Server (JSON-RPC request)
    {
        "jsonrpc": "2.0",
        "method": "tools/call",
        "params": {
            "name": "search_database",
            "arguments": {"name": "John"}
        }
    }
    ↓
MCP Server: Connects to PostgreSQL
    ↓
PostgreSQL: Returns [John Doe, john@example.com, ...]
    ↓
MCP Server → MCP Client (JSON response)
    {
        "result": {
            "content": [{
                "type": "text",
                "text": "Found: John Doe, john@example.com"
            }]
        }
    }
    ↓
Claude: "I got the results. Here's the answer..."

Server vs Client

You Build (MCP Server):
• Python/JavaScript application
• Listens for requests
• Calls your database/API
• Returns results

Claude Provides (MCP Client):
• Claude Desktop app
• Claude API
• LangGraph/AutoGen integrations
• Automatically sends requests

Communication Protocol

  • Transport: Stdio (process pipes) or HTTP/WebSocket
  • Format: JSON-RPC 2.0 (standardized)
  • Tools: Functions your server exposes
  • Resources: Data your server provides
Why JSON-RPC? It's language-independent, standardized, and every tool understands it. Write your MCP in Python, use it with JavaScript—doesn't matter.

Key Concepts

1. Tools

Functions that your MCP server exposes. Claude can call them.

What is a Tool?
Tool: query_database
Description: "Search the database for customers"
Parameters:
  - name: "John" (required)
  - limit: 10 (optional)

Result: List of customers matching "John"
Think of tools as methods your server offers. Claude asks "Can I call query_database?" and your server says "Yes, here's what it does and what parameters it needs."

2. Resources

Static data your server provides (files, documents, knowledge bases).

  • URI-based identifiers (like file paths)
  • Claude can read them without calling functions
  • Useful for context, documents, static information

3. Server Discovery

When Claude connects to your MCP server, it asks: "What tools and resources do you have?"

Server Lists What It Can Do
Claude: "What can you do?"

MCP Server responds:
{
    "tools": [
        {"name": "query_database", "description": "..."},
        {"name": "insert_record", "description": "..."},
        {"name": "delete_record", "description": "..."}
    ],
    "resources": [
        {"uri": "file:///data/customers.csv", "description": "..."}
    ]
}

Claude: "Great! I can call these 3 tools and access this file."

4. Tool Calling Flow

1
Discovery

Claude asks MCP server: "What tools do you have?"

2
Request

Claude decides to call a tool with parameters

3
Execution

MCP server executes the tool (calls database, API, etc.)

4
Response

MCP server returns results to Claude

5
Reasoning

Claude uses results to answer the user's question

5. Input Validation

Each tool declares what parameters it accepts (schema).

Tool Schema
Tool: search_database

Input Schema:
{
    "type": "object",
    "properties": {
        "name": {
            "type": "string",
            "description": "Customer name to search for"
        },
        "limit": {
            "type": "integer",
            "description": "Max results (default: 10)"
        }
    },
    "required": ["name"]
}

Valid call: search_database(name="John", limit=5) ✓
Invalid call: search_database() ✗ (missing required 'name')

6. Error Handling

Tools return either results or errors.

Tool Results
Success:
{
    "content": [{
        "type": "text",
        "text": "Found 3 customers named John"
    }],
    "isError": false
}

Error:
{
    "content": [{
        "type": "text",
        "text": "Database connection failed"
    }],
    "isError": true
}
Key Point: Every tool response has 2 fields: content (what happened) and isError (success or failure). Claude handles both cases.

Python MCP Boilerplate

Python - Full Template
"""
MCP Server Template
A minimal MCP server with tools and error handling.
"""

import asyncio
import json
from mcp.server import Server
from mcp.types import Tool, TextContent, ToolResult

# ============================================================================
# 1. CREATE THE SERVER
# ============================================================================

server = Server("my-server")  # Replace with your server name

# ============================================================================
# 2. DEFINE YOUR TOOLS
# ============================================================================

@server.tool()
async def my_first_tool(param1: str, param2: int = 10) -> ToolResult:
    """
    This is what your tool does.
    
    Args:
        param1: A required string parameter
        param2: An optional integer (default: 10)
    
    Returns:
        A ToolResult with content
    """
    try:
        # Your business logic here
        result = f"Processed: {param1} with {param2}"
        
        return ToolResult(
            content=[TextContent(text=result)],
            is_error=False
        )
    except Exception as e:
        return ToolResult(
            content=[TextContent(text=f"Error: {str(e)}")],
            is_error=True
        )

@server.tool()
async def my_second_tool(data: str) -> ToolResult:
    """Another tool example"""
    try:
        # Process the data
        processed = data.upper()
        return ToolResult(content=[TextContent(text=processed)])
    except Exception as e:
        return ToolResult(
            content=[TextContent(text=f"Error: {str(e)}")],
            is_error=True
        )

# ============================================================================
# 3. REGISTER TOOLS WITH MCP
# ============================================================================

server.register_tool(
    Tool(
        name="my_first_tool",
        description="Describe what this tool does here",
        inputSchema={
            "type": "object",
            "properties": {
                "param1": {
                    "type": "string",
                    "description": "What is param1?"
                },
                "param2": {
                    "type": "integer",
                    "description": "What is param2?",
                    "default": 10
                }
            },
            "required": ["param1"]
        }
    )
)

server.register_tool(
    Tool(
        name="my_second_tool",
        description="Describe what this tool does here",
        inputSchema={
            "type": "object",
            "properties": {
                "data": {
                    "type": "string",
                    "description": "The data to process"
                }
            },
            "required": ["data"]
        }
    )
)

# ============================================================================
# 4. START THE SERVER
# ============================================================================

if __name__ == "__main__":
    async def main():
        async with server:
            print("MCP Server is running...")
            await asyncio.sleep(float('inf'))
    
    asyncio.run(main())

With Database Connection (PostgreSQL)

Python - Database MCP
"""
MCP Server with Database Connection
Connects to PostgreSQL and exposes database operations as tools.
"""

import asyncio
import psycopg2
from psycopg2.extras import RealDictCursor
from mcp.server import Server
from mcp.types import Tool, TextContent, ToolResult

server = Server("database-mcp")

# ============================================================================
# DATABASE CONNECTION POOL
# ============================================================================

def get_db_connection():
    """Get connection to PostgreSQL"""
    try:
        conn = psycopg2.connect(
            host="localhost",
            database="mydb",
            user="postgres",
            password="password"
        )
        return conn
    except Exception as e:
        raise Exception(f"Database connection failed: {str(e)}")

# ============================================================================
# DATABASE TOOLS
# ============================================================================

@server.tool()
async def query_database(sql: str, limit: int = 10) -> ToolResult:
    """Execute a SELECT query on the database"""
    try:
        conn = get_db_connection()
        cursor = conn.cursor(cursor_factory=RealDictCursor)
        
        # Limit the query to prevent abuse
        safe_sql = f"{sql} LIMIT {limit}"
        cursor.execute(safe_sql)
        
        rows = cursor.fetchall()
        cursor.close()
        conn.close()
        
        # Format as readable text
        if not rows:
            return ToolResult(content=[TextContent(text="No results found")])
        
        result_text = "\n".join([str(dict(row)) for row in rows])
        return ToolResult(content=[TextContent(text=result_text)])
    
    except Exception as e:
        return ToolResult(
            content=[TextContent(text=f"Query failed: {str(e)}")],
            is_error=True
        )

@server.tool()
async def insert_record(table: str, data: dict) -> ToolResult:
    """Insert a record into the database"""
    try:
        conn = get_db_connection()
        cursor = conn.cursor()
        
        columns = ", ".join(data.keys())
        placeholders = ", ".join(["%s"] * len(data))
        values = list(data.values())
        
        sql = f"INSERT INTO {table} ({columns}) VALUES ({placeholders})"
        cursor.execute(sql, values)
        
        conn.commit()
        cursor.close()
        conn.close()
        
        return ToolResult(
            content=[TextContent(text=f"Record inserted into {table}")]
        )
    
    except Exception as e:
        return ToolResult(
            content=[TextContent(text=f"Insert failed: {str(e)}")],
            is_error=True
        )

# ============================================================================
# REGISTER TOOLS
# ============================================================================

server.register_tool(
    Tool(
        name="query_database",
        description="Execute a SELECT query on PostgreSQL",
        inputSchema={
            "type": "object",
            "properties": {
                "sql": {
                    "type": "string",
                    "description": "SQL SELECT query"
                },
                "limit": {
                    "type": "integer",
                    "description": "Maximum rows to return (default: 10)"
                }
            },
            "required": ["sql"]
        }
    )
)

server.register_tool(
    Tool(
        name="insert_record",
        description="Insert a new record into the database",
        inputSchema={
            "type": "object",
            "properties": {
                "table": {
                    "type": "string",
                    "description": "Table name"
                },
                "data": {
                    "type": "object",
                    "description": "Column-value pairs to insert"
                }
            },
            "required": ["table", "data"]
        }
    )
)

# ============================================================================
# START SERVER
# ============================================================================

if __name__ == "__main__":
    async def main():
        async with server:
            print("Database MCP Server running...")
            await asyncio.sleep(float('inf'))
    
    asyncio.run(main())

With REST API Integration

Python - API MCP
"""
MCP Server with REST API Integration
Connects to external APIs and exposes them as tools.
"""

import asyncio
import httpx
from mcp.server import Server
from mcp.types import Tool, TextContent, ToolResult

server = Server("api-mcp")

# ============================================================================
# API TOOLS
# ============================================================================

@server.tool()
async def call_external_api(endpoint: str, method: str = "GET", 
                            data: dict = None) -> ToolResult:
    """Call an external REST API"""
    try:
        async with httpx.AsyncClient() as client:
            url = f"https://api.example.com/{endpoint}"
            
            if method == "GET":
                response = await client.get(url)
            elif method == "POST":
                response = await client.post(url, json=data)
            else:
                return ToolResult(
                    content=[TextContent(text=f"Unsupported method: {method}")],
                    is_error=True
                )
            
            result = response.json()
            return ToolResult(content=[TextContent(text=str(result))])
    
    except Exception as e:
        return ToolResult(
            content=[TextContent(text=f"API call failed: {str(e)}")],
            is_error=True
        )

@server.tool()
async def get_weather(city: str) -> ToolResult:
    """Get weather for a city (example API integration)"""
    try:
        async with httpx.AsyncClient() as client:
            response = await client.get(
                f"https://api.openweathermap.org/data/2.5/weather",
                params={"q": city, "appid": "YOUR_API_KEY"}
            )
            data = response.json()
            
            result = f"Weather in {city}: {data['weather'][0]['main']}, {data['main']['temp']}K"
            return ToolResult(content=[TextContent(text=result)])
    
    except Exception as e:
        return ToolResult(
            content=[TextContent(text=f"Weather fetch failed: {str(e)}")],
            is_error=True
        )

# ============================================================================
# REGISTER TOOLS
# ============================================================================

server.register_tool(
    Tool(
        name="call_external_api",
        description="Call an external REST API",
        inputSchema={
            "type": "object",
            "properties": {
                "endpoint": {
                    "type": "string",
                    "description": "API endpoint path"
                },
                "method": {
                    "type": "string",
                    "enum": ["GET", "POST"],
                    "description": "HTTP method"
                },
                "data": {
                    "type": "object",
                    "description": "Data for POST request"
                }
            },
            "required": ["endpoint"]
        }
    )
)

server.register_tool(
    Tool(
        name="get_weather",
        description="Get current weather for a city",
        inputSchema={
            "type": "object",
            "properties": {
                "city": {
                    "type": "string",
                    "description": "City name"
                }
            },
            "required": ["city"]
        }
    )
)

# ============================================================================
# START SERVER
# ============================================================================

if __name__ == "__main__":
    async def main():
        async with server:
            print("API MCP Server running...")
            await asyncio.sleep(float('inf'))
    
    asyncio.run(main())

Creating Tools

Tool Structure

Every tool has 3 parts:

1
Function Definition

The actual code that does the work

2
Description

What the tool does (for Claude to understand)

3
Input Schema

What parameters the tool accepts

Simple Tool Template

Pattern
@server.tool()
async def my_tool(param1: str, param2: int = 10) -> ToolResult:
    """
    One-line description of what this tool does.
    
    Args:
        param1: Description of param1
        param2: Description of param2 (with default)
    """
    try:
        # Your code here
        result = do_something(param1, param2)
        
        return ToolResult(
            content=[TextContent(text=str(result))],
            is_error=False
        )
    except Exception as e:
        return ToolResult(
            content=[TextContent(text=f"Error: {str(e)}")],
            is_error=True
        )

# Register it
server.register_tool(
    Tool(
        name="my_tool",  # Tool ID (what Claude calls)
        description="What this tool does",  # For Claude
        inputSchema={  # Parameters
            "type": "object",
            "properties": {

Database Query

Database Tool
@server.tool()
async def query_database(query: str) -> ToolResult:
    """Execute a database query"""
    try:
        conn = get_db_connection()
        results = conn.execute(query).fetchall()

            is_error=True

API Tool
@server.tool()
async def fetch_api(url: str, method: str = "GET") -> ToolResult:
    """Call an external API"""
    try:
        async with httpx.AsyncClient() as client:
            response = await client.request(method, url)
            return ToolResult(content=[
                TextContent(text=response.json())
            ])
    except Exception as e:
        return ToolResult(
            content=[TextContent(text=f"Error: {str(e)}")],
            is_error=True
        )

File Operation

File Tool
@server.tool()
async def read_file(path: str) -> ToolResult:
    """Read a file"""
    try:
        with open(path, 'r') as f:
            content = f.read()

    except Exception as e:

            content=[TextContent(text=f"Error: {str(e)}")],
            is_error=True
        )

Best Practices

  • Always wrap in try/except
  • Always return ToolResult
  • Always provide clear description
  • Always validate input
  • Always set is_error correctly
  • Never expose sensitive data in errors
  • Never make tools do too much
Tool Design Rule: One tool = one function. Don't make mega-tools that do multiple things. Keep them focused.

Using MCPs with AI Agents

LangGraph Integration

Python - LangGraph + MCP
from langgraph.graph import StateGraph, START, END
from langchain_anthropic import ChatAnthropic
from langchain_core.tools import tool
from langchain_core.messages import HumanMessage
import subprocess
import json

# ============================================================================
# WRAPPER TOOL: Calls your MCP
# ============================================================================

@tool
def database_tool(query: str) -> str:
    """Search the database using MCP"""
    
    # Call your MCP server
    request = {
        "jsonrpc": "2.0",
        "method": "tools/call",
        "params": {
            "name": "query_database",
            "arguments": {"query": query}
        }
    }
    
    result = subprocess.run(
        ["python", "my_mcp_server.py"],
        input=json.dumps(request),
        capture_output=True,
        text=True
    )
    
    return result.stdout

# ============================================================================
# CREATE AGENT
# ============================================================================

class AgentState:
    messages: list

llm = ChatAnthropic(model="claude-opus-4-6")
tools = [database_tool]
llm_with_tools = llm.bind_tools(tools)

# ============================================================================
# AGENT LOGIC
# ============================================================================

def agent_step(state):
    """Agent reasoning step"""
    response = llm_with_tools.invoke(state["messages"])
    return {"messages": state["messages"] + [response]}

# ============================================================================
# BUILD GRAPH
# ============================================================================

graph = StateGraph(AgentState)
graph.add_node("agent", agent_step)
graph.add_edge(START, "agent")
graph.add_edge("agent", END)

agent = graph.compile()

# ============================================================================
# RUN AGENT
# ============================================================================

result = agent.invoke({
    "messages": [HumanMessage(content="Find customers named John")]
})

print(result)

2. Error Handling

Always Handle Errors
@server.tool()
async def my_tool(param: str) -> ToolResult:
    """Tool with proper error handling"""
    try:
        # Your code
        if not param:
            # Validate input
            return ToolResult(
                content=[TextContent(text="Parameter cannot be empty")],
                is_error=True
            )
        
        result = do_work(param)
        return ToolResult(content=[TextContent(text=result)])
    
    except ValueError as e:
        # Specific error
        logger.error(f"Invalid value: {str(e)}")
        return ToolResult(
            content=[TextContent(text=f"Invalid input: {str(e)}")],
            is_error=True
        )
    
    except DatabaseError as e:
        # Database error
        logger.error(f"Database error: {str(e)}")
        return ToolResult(
            content=[TextContent(text="Database operation failed")],
            is_error=True
        )
    
    except Exception as e:
        # Generic fallback
        logger.error(f"Unexpected error: {str(e)}", exc_info=True)
        return ToolResult(
            content=[TextContent(text="An error occurred")],
            is_error=True
        )

Troubleshooting

Common Issues & Solutions

Issue: MCP not showing in Claude Desktop

Problem: Tools aren't available in Claude

Solutions:
1. Check config file syntax (valid JSON?)
2. Verify file path is absolute, not relative
3. Restart Claude Desktop completely
4. Check server is actually running (try running manually)
5. Look at Claude logs for errors

Issue: Tool call fails silently

Problem: Tool returns empty result

Solutions:
1. Add logging to see what's happening
2. Test tool directly: python my_mcp_server.py
3. Check database/API connection
4. Verify parameters are being passed correctly
5. Add print() statements for debugging

Issue: "Tool not found" error

Problem: Claude says tool doesn't exist

Solutions:
1. Check tool name matches exactly (case-sensitive)
2. Verify tool is registered: server.register_tool()
3. Make sure @server.tool() decorator is present
4. Check input schema is valid JSON

Issue: Database connection fails

Problem: "Could not connect to database"

Solutions:
1. Is database actually running?
2. Verify credentials (host, user, password)
3. Check firewall allows connection
4. Try connecting directly with psql/mysql
5. Check environment variables are set

Issue: Performance is slow

Problem: Tool takes too long to respond

Solutions:
1. Add database indexes
2. Implement result caching
3. Limit result set size (use LIMIT in SQL)
4. Profile tool execution time
5. Consider async processing
