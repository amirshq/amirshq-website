🤖
AGENT
The whole chatbot
top level
Agent is the entire chatbot — your project container. One agent = one bot (e.g., "PizzaBot" or "BankBot").

It holds everything: flows, intents, entities, webhooks. You configure its language, timezone, and default welcome message here.

Think of it like a software application — the agent is the app, everything else lives inside it.
📊
FLOWS
Topic modules inside the agent
modules
Flows are like chapters in a book, or modules in a codebase. Each flow handles one major topic area.

Example: Billing Flow, Order Flow, Support Flow.

Every agent automatically has a Default Start Flow — this is where every conversation begins. Flows can call other flows, like function calls. This keeps large agents clean and organized.
📄
PAGES
The current state of the conversation
★ most important
Pages are the most important concept in Dialogflow CX. A page = the current state of the conversation.

At any moment, the bot is on exactly one page. Like a screen in an app — you're always on one screen at a time.

Each page can:
• Say something when you arrive (Entry Fulfillment)
• Collect information from the user (Form)
• Decide where to go next (Routes)

Example pages in a pizza bot: Greeting → Choose Size → Choose Toppings → Confirm Order
🎯
INTENTS
What the user wants to do
NLU layer
Intents classify what the user means. You train each intent with example phrases — the NLU model learns to recognize similar inputs.

"I want a pizza" → matches intent: order.pizza
"Cancel my order" → matches intent: cancel.order

Intents don't do anything by themselves — they trigger Routes. A route says "when this intent fires on this page → do this action."

Key insight: The same intent can have different behavior on different pages. "Yes" on the order page confirms an order. "Yes" on the cancel page confirms cancellation.
📦
ENTITIES
Pieces of data extracted from user input
extraction
Entities pull structured data out of free-form text.

From: "I want a large pepperoni pizza"
Extracts: pizza-size = "large" and topping = "pepperoni"

Three types:
• System entities — built-in: @sys.date, @sys.number, @sys.city
• Custom entities — you define: @pizza-size = small/medium/large
• Regex entities — patterns: @order-id = BK[0-9]{5}

Extracted entity values become Parameters stored in the session.
⚡
FULFILLMENT
What happens when a route fires
action layer
Fulfillment is the action that runs when a route is triggered. It can:

• 💬 Send a text/card response to the user
• 🌐 Call a Webhook (your backend API)
• 📝 Set parameters (store data in the session)
• ↩️ Transition to another page or flow

Multiple actions can happen at once. Example: call webhook → set parameter → send response → transition to next page.

This is where your business logic lives.
How they all connect
Agent contains Flows which contain Pages which have Routes
Routes are triggered by Intents (which use Entities to extract data)
When a route fires → Fulfillment runs (responses, webhooks, transitions)
